The cold-start tax

Every AI tool starts blank.

Claude, ChatGPT, Cursor, Copilot — each one boots without memory of what you were working on. So every prompt re-explains the project: "the file I just edited", "the test that failed", "the branch you read last time". 76% of developers use ≥ 2 AI tools daily. The re-explanation tax is real.

What Ithuriel is

One prompt, full execution.

A macOS-native agent that lives in your menu bar, watches your workspace, and uses Google Gemini to drive your keyboard, mouse, file system, and apps — with the project's full context already loaded into every turn.

Why the agent never starts cold

Continuous context capture.

Five subsystems run quietly in the background. Every agent turn is seeded with a fresh ContextSnapshot.

  • FSEventStream — 5s-debounced file changes
  • NSWorkspace — active editor + open tabs
  • GitCapture — branch, recent commits, diff
  • TerminalCapture — last 20 shell commands
  • Redactor — scrubs API keys, .env, .ssh/

Safety

Permission-cheap, kill-switch-cheap.

1
Sandboxed file ops. read/write/delete pinned to the active workspace. Outside? Refused.
2
Secrets scrubber. sk-, ghp_, AIza, xoxb-, api_key=… stripped before anything leaves the Mac.
3
Destructive prompts. write_file / delete_file / run_shell / quit_app can require explicit confirmation.
4
Lazy permissions. Onboarding only asks for what isn't already granted — no nagging.
5
Global kill switch. ⌃ ⌥ ⌘ . stops the agent mid-step. Always available.
Privacy that survives shipping

The agent is dangerous by design. So every off-Mac payload is funnelled through the Redactor first; every destructive action is gated on user policy; every sensitive path (.env, .ssh/, secrets/) is refused even inside the sandbox.

Local-only mode short-circuits every network call. Promise it, mean it.
